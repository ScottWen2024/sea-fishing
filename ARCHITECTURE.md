# 海钓通 技术架构

---

## 一、双端架构

```
┌──────────────────────────────────────────────────────┐
│                    海钓通                             │
│                                                      │
│  ┌──────────────┐              ┌──────────────┐      │
│  │   H5 原型     │              │  微信小程序    │      │
│  │ index.html   │              │ fishingMNew/ │      │
│  │ 170KB 单文件 │              │ 11 pages     │      │
│  └──────┬───────┘              └──────┬───────┘      │
│         │ localStorage                 │ 云数据库     │
│         │                              │ cloud1      │
└─────────┼──────────────────────────────┼─────────────┘
          │                              │
    ┌─────▼─────┐                  ┌─────▼─────┐
    │ 和风天气API │                  │  云函数    │
    │ 高德地图API │                  │  代理API   │
    └───────────┘                  └───────────┘
```

## 二、数据流

```
用户操作 → 前端渲染 → API 调用 → 数据展示
                │
                ├── H5：直接调 fetch() 调用和风/高德 API
                │       失败时降级为本地模拟数据
                │
                └── 小程序：通过云函数代理 API
                         云函数内置 30 分钟缓存
                         前端调 wx.cloud.database() 读写
```

## 五、关键技术决策

| 决策 | 理由 |
|------|------|
| H5 用单文件 | 快速原型验证，不改构建工具 |
| 小程序暂未抽组件 | 功能稳定后再重构 |
| localStorage → 云数据库 | 迁移时云函数做同步 |
| 云函数代理 API | 隐藏密钥、支持缓存、减少前端请求 |
| 三环境策略 | 原型验证 → UAT测试 → 生产发布 |
| Deep Waters 配色 | 海事主题，专业感 |

## 六、外部依赖

| 服务 | 用途 | 替代方案 |
|------|------|------|
| 和风天气 API | 天气/潮汐数据 | 降级为蒲福风级模拟数据 |
| 高德地图 JS API | 钓点地图标注 | 静态列表替代 |
| 腾讯云 IM | 聊天功能 | 暂不启用 |
| Jest | 单元测试 | 无替代 |
