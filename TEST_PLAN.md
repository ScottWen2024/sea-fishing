# 海钓通 全面测试计划

---

## 一、测试范围定义

### 什么要测

```
优先级    测试对象              原因
──────────────────────────────────────────
P0   ⭐⭐⭐ 核心计算逻辑         错一个会影响所有用户
P1   ⭐⭐   API 调用链路         API 挂了要有降级，不能白屏
P1   ⭐⭐   关键页面流程         约钓/预约/渔获主链路不能断
P2   ⭐     边界和异常           罕见场景但出了就是事故
P3   ⭐     样式和交互           不影响功能，但影响体验
```

### 什么不测

| 不测 | 原因 |
|------|------|
| DevTools 编译流程 | 微信平台的事，测不了 |
| 和风天气数据准确性 | 第三方的事 |
| 小程序审核流程 | 微信审核团队的事 |
| 用户手机兼容性 | MVP 阶段先不管 |

## 二、P0：核心计算逻辑

### 2.1 天气/环境计算

| 函数 | 为什么关键 | 测什么 |
|------|------|------|
| `estimateWave()` | 浪高判断影响出海安全建议 | 4 个风级区间 + 边界值(恰好10/20/30/40) + 负数/非数字 |
| `getWeatherAdvice()` | 全站出海建议都依赖它 | 3 种等级 + 边界值(恰好3级/4级/6级) + 字符串"1-3"级 |
| `calcFishingIndex()` | 首页核心功能，钓友决策依据 | 满分/零分/中等 + 缺失数据 + 空输入 + 单项满分组合 |

### 2.2 距离/位置计算

| 函数 | 为什么关键 | 测什么 |
|------|------|------|
| `haversine()` | GPS 定位找最近港口 | 同点→0、短距(<1km)、中距(同城)、长距(跨省) + 极端纬度(赤道/北极) + 反向参数 |

### 2.3 月相计算

| 函数 | 为什么关键 | 测什么 |
|------|------|------|
| `getMoonPhase()` | 钓鱼指数中月相维度 + 月相日历 | 已知新月日、已知满月日、随机日 + 跨年边界 + 输入格式错误 |

### 2.4 综合评分

| 函数 | 为什么关键 | 测什么 |
|------|------|------|
| `getRecIndex()` | 船长推荐分影响排序 | 满分条件、最低分条件、空数组/ undefined 防护 |

## 三、P1：API 调用链路

### 3.1 天气 API

| 测试场景 | 怎么测 | 预期结果 |
|------|------|------|
| 正常返回 200 | 用真实 key 调一次 | code=200, daily 数组有 7 天数据 |
| 返回 403 | 用过期 key | 降级为模拟数据，页面不崩溃 |
| 网络超时 | 断网后刷新 | 3 秒超时→降级数据→显示"模拟数据"标记 |
| 返回数据缺少字段 | Mock 一个缺字段的响应 | 缺字段显示"--"，不报错 |
| 15 天预测可用 | 用真实 key 调 15d 接口 | 返回 15 天数据，字段完整 |

### 3.2 潮汐 API

| 测试场景 | 怎么测 | 预期结果 |
|------|------|------|
| 正常返回 | 用 P2970 站点调一次 | 4 条潮汐记录 + 24 条小时数据 |
| 站点不存在 | 用不存在站点 ID | 降级，潮汐栏不显示 |
| 网络超时 | 断网后刷新 | 不影响天气卡片显示 |

### 3.3 高德地图加载

| 测试场景 | 怎么测 | 预期结果 |
|------|------|------|
| 正常加载 | 点地图按钮 | 显示地图+20 个港口标注 |
| JS 加载失败 | 断网后点地图 | 显示"地图加载失败"提示，不崩溃 |
| GPS 定位被拒 | 浏览器禁止定位 | 显示"请允许定位权限"提示 |

## 四、P1：关键页面流程

### 4.1 约钓主链路

```
首页点「📢 约钓」 → 填写信息 → 提交
→ 约钓列表出现 → 其他人刷新也能看到
```

| 测试点 | 怎么测 | 预期 |
|------|------|------|
| 空白提交 | 不填日期→点提交 | 提示"请填写日期" |
| 正常提交 | 填完所有字段→提交 | 列表出现新条目 |
| 最大人数边界 | 填 0 人/100 人 | 应限制 2-20 |

### 4.2 找船-预约链路

| 测试点 | 怎么测 | 预期 |
|------|------|------|
| 展开/收起 | 点"需要找船长"→再点"收起" | 展开显示船长，收起隐藏 |
| 拼船/导钓切换 | 点导钓 | 只显示有导钓服务的船长 |
| 港口筛选 | 选"南澳岛" | 只显示南澳岛的船长 |

### 4.3 渔获记录链路

| 测试点 | 怎么测 | 预期 |
|------|------|------|
| 模拟识鱼 | 点拍照按钮 | 显示"黑鲷"结果 |
| 记录渔获 | 点"记录这条渔获" | 时间线新增一条，统计面板数字+1 |
| 大鱼排行 | 切换到"大鱼排行" | 按重量排序，金银铜牌正确 |
| 空状态 | 删除所有渔获后看页面 | 显示"还没有渔获记录" |

## 五、P2：边界和异常

### 5.1 数据边界

| 场景 | 输入 | 预期 |
|------|------|------|
| 港口列表为空 | PORT_DATA = [] | 首页不崩溃，港口选择器显示空 |
| 船长列表为空 | 没有船长数据 | 约钓页展开后显示"暂无船长" |
| 渔获 0 条 | 新用户 | 渔获页显示空状态引导 |
| 拼船 0 条 | 新用户 | 首页和约钓页显示"暂无约钓" |
| 超大量数据 | 100 条拼船 | 页面滚动流畅不卡 |
| 评分无数据 | reviews = [] | 推荐分公式不报错 |

### 5.2 用户操作边缘

| 场景 | 怎么测 | 预期 |
|------|------|------|
| 快速双击按钮 | 连点两次发起约钓 | 只创建一次，不会重复 |
| 输入超长文本 | 目标鱼种填 500 个字 | 显示截断或限制长度 |
| 日期选过去 | 选昨天的日期发起约钓 | 应提示"不能选过去的日期" |
| 连续切换 Tab | 快速点底部 Tab | 不卡顿，不白屏 |
| 同时打开两个窗口 | 两个浏览器标签 | 不会数据冲突 |

### 5.3 环境异常

| 场景 | 怎么测 | 预期 |
|------|------|------|
| localStorage 损坏 | 手动改坏 localStorage | 重置为默认数据，不崩溃 |
| 和风 API 全挂 | 断网 + 没缓存 | 显示模拟数据，标注"模拟数据" |
| 图片加载失败 | 不存在的照片 URL | 显示占位图，不报错 |

## 六、测试分级执行时间表

| 阶段 | 触发条件 | 测试内容 | 预计耗时 |
|------|------|------|:---:|
| **开发自测** | 每次改完代码 | P0 核心函数 + 改了哪个页面就测哪个 | 2 分钟 |
| **提交前测试** | git commit 前 | P0 全部 + P1 关键链路 | 5 分钟 |
| **集成测试** | 每日首次编译前 | P0 + P1 + P2 随机抽 3 项 | 10 分钟 |
| **UAT 回归** | 发版前 | 全部 checklist(27 项) + P0 自动化测试 | 15 分钟 |
| **生产验证** | 发版后 | P1 关键链路在生产环境走一遍 | 5 分钟 |

## 七、测试数据管理

### 7.1 需要准备的测试数据

| 数据 | 位置 | 内容 |
|------|------|------|
| 天气 Mock | `tests/fixtures/weather.json` | 4 种典型天气（完美/好/中等/恶劣） |
| 潮汐 Mock | `tests/fixtures/tide.json` | 正常潮汐 + 空数据 |
| 船长 Mock | 测试代码内 | 5 种类型的船长（满分配置/新船长/无导钓/无评分/空数据） |
| 渔获 Mock | 测试代码内 | 0 条/3 条/50 条渔获 |
| 拼船 Mock | 测试代码内 | 进行中/已满/空列表 |

### 7.2 测试数据原则

- **不依赖生产数据**：所有测试数据都是构造的，不会影响真实用户
- **可重复执行**：每次跑测试结果一致
- **覆盖极端值**：除了正常值，必须有最大值/最小值/空值/错误值

| 指标 | 当前 | 目标 | 怎么算 |

## 九、当前待补充的测试（优先级排序）

1. ⬜ **边界值测试** — estimateWave(10)、getWeatherAdvice(3)、getWeatherAdvice("1-3"级)
2. ⬜ **极端输入测试** — 空数组、undefined、负数、超大数据
3. ⬜ **API 降级测试** — 和风返回 403 时页面是否显示模拟数据
4. ⬜ **快速双击测试** — 发起约钓按钮连点两次是否创建重复
5. ⬜ **H5 + 小程序同步回归** — 改了一端，另一端也要过 checklist
