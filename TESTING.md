# 海钓通 测试分层方案

---

## 一、三层测试金字塔

```
        ╱  E2E  ╲          ← 全流程（目前不做）
       ╱  UAT   ╲          ← checklist 手工回归
      ╱ 集成测试 ╲          ← API + 云函数 + 页面跳转
     ╱  单元测试  ╲         ← 纯函数（天气计算、钓鱼指数）
    ─────────────────
```

## 二、各层详情

### 1. 单元测试（核心 = 可以自动化）

**测什么：** 纯逻辑函数，不依赖 DOM / 网络 / 数据库

| 函数 | 位置 | 输入 → 期望输出 |
|------|------|------|
| `haversine(lat1,lng1,lat2,lng2)` | H5 JS | 已知两点 → 已知距离 |
| `estimateWave(windSpeed)` | H5 JS | 风速 10km/h → "0.1-0.3m" |
| `getWeatherAdvice(windScale)` | H5 JS | 3 级 → {level:"good"} |
| `getMoonPhase(dateStr)` | H5 JS | 2026-05-08 → {icon:"🌑",isSpring:true} |
| `calcFishingIndex(dayData,tideData)` | H5 JS | 输入 → 0-100 分 |
| `getRecIndex(captain)` | H5 JS | 船长对象 → 推荐分 |
| `formatDate(dateStr)` | H5 JS | "2026-06-01" → "6月1日" |

**测多少：** 7 个函数，每个 2-3 条用例，一共约 20 条

**用什么：** Jest（npm install -g jest）

**怎么跑：**
```bash
cd D:\Fishing project\tests
npx jest
```

**示例用例：**
```javascript
// tests/weather.test.js
describe('estimateWave', () => {
  test('风速≤10 → 0.1-0.3m', () => {
    expect(estimateWave(5)).toBe('0.1-0.3m');
  });
  test('风速20-30 → 0.6-1.2m', () => {
    expect(estimateWave(25)).toBe('0.6-1.2m');
  });
  test('风速>40 → >2.0m', () => {
    expect(estimateWave(50)).toBe('>2.0m');
  });
});
```

### 2. 集成测试（手工 + 半自动）

| 测试项 | 怎么测 | 频率 |
|------|------|:---:|
| 天气 API 正常返回 | curl 调一次看返回码 | 改代码后 |
| 潮汐 API 正常返回 | curl 调一次看返回码 | 改代码后 |
| 高德地图能加载 | H5 点地图按钮看是否显示 | 改代码后 |
| 云函数 getUserSig 能调用 | DevTools 云端测试 | 改云函数后 |
| 云数据库能读写 | 小程序创建约钓看是否成功 | 每次提交前 |
| H5 → 小程序 页面跳转一致 | 两边点一遍所有按钮 | 每次改导航后 |

### 3. UAT 测试（手工 checklist）

**就是 checklist.md 的 27 项。** 每次发版前完整走一遍。

| 场景 | 步骤 | 预期 |
|------|------|------|
| 新用户打开首页 | 清缓存 → 打开 | 天气加载、鱼经显示、约钓列表为空 |
| 发起约钓 | 首页点约钓 → 填表 → 提交 | 列表中显示、其他人刷新能见 |
| 拍照识鱼 | 渔获页点拍照 → 模拟识别 | 结果展示、点记录 → 时间线更新 |
| 找船-预约 | 约钓页展开找船 → 选船长 → 没预约按钮（功能未完成）|

## 三、谁来测、什么时候测

| 阶段 | 谁测 | 测什么 | 频率 |
|------|------|------|:---:|
| **开发时** | AI（写代码时自己脑子过） | 逻辑是否正确 | 每次写 |
| **提交前** | 你 | 点击流程是否通 | 每次 git commit |
| **提审前** | 你 | checklist 全部走一遍 | 发版前 |
| **上线后** | 真实用户 | 实际使用反馈 | 持续 |

## 四、现在就可以做的

1. **单元测试** — 从 H5 抽出 7 个纯函数 → 写 20 条用例 → npm test

2. **checklist 扩展** — 加上云开发相关项：
   - [ ] 发起约钓写入云数据库
   - [ ] 另一台设备能看到新约钓
   - [ ] 加入约钓成功统计更新

3. **错误场景测试**：
   - [ ] 网络断开时页面不崩溃
   - [ ] 和风 API 返回 403 时显示模拟数据
   - [ ] 高德地图加载失败时不卡死

## 五、测试覆盖率目标

| 层级 | 当前覆盖率 | 目标 |
|------|:---:|:---:|
| 核心函数单元测试 | 0% | **50%**（7 个函数 × 20 条用例） |
| 集成测试 | 0% | 核心 API 两根都过 |
