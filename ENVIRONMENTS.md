# 海钓通 多环境管理方案

---

## 一、环境定义

| 环境 | 代码位置 | 数据库 | 用途 | 用户 |
|------|------|------|------|------|
| **原型 H5** | `D:\Fishing project\index.html` | localStorage | 快速验证想法、UI 试用 | 只有你 |
| **UAT 小程序** | `D:\Fishing project\fishingMNew\` | 云环境 `cloud1` | 功能测试、内部试用 | 你 + 少数测试用户 |
| **生产 小程序** | 同代码、不同云环境 | 云环境 `cloud-prod`（待建） | 正式发布 | 所有用户 |

## 二、三环境的差异

```
原型 H5                UAT 小程序            生产 小程序
─────────            ──────────            ──────────
单文件 170KB          11 个页面文件          同 UAT 代码
localStorage         云数据库 cloud1        云数据库 cloud-prod
无后端              云函数                云函数
直接调 API          云函数代理 API         云函数代理 API
任何人都打不开       扫码体验版            正式发布
```

## 三、代码流转

```
  H5 原型                UAT 小程序              生产 小程序
  ┌───────┐            ┌───────────┐          ┌───────────┐
  │ 新功能  │ ──验证──→ │ 迁移到WXML │ ──测试──→ │ 提审发布  │
  │ 快速试  │          │ 接云数据库  │         │           │
  └───────┘            └───────────┘          └───────────┘
      ↑                      ↑                     ↑
  改了就刷新              DevTools 编译         微信审核通过
```

### 标准流程

1. **H5 验证**：新想法 → H5 改了 → 浏览器看效果 → 满意进入下一步
2. **UAT 迁移**：H5 代码 → 改写成 WXML/WXSS/JS → DevTools 编译 → 你自己扫码测试
3. **UAT 测试**：云数据库 `cloud1` 上跑完整流程 → 约钓/预约/渔获 → 确认无 bug
4. **生产上线**：代码不变，只把云环境从 `cloud1` 切成 `cloud-prod` → DevTools 上传 → 微信后台提交审核 → 审核通过 → 发布

## 四、数据库分离（核心！）

| 数据 | cloud1（UAT） | cloud-prod（生产） |
|------|:---:|:---:|
| 测试拼船 | ✅ 随便造 | ❌ 不能污染 |
| 假船长 | ✅ | ❌ |
| 真实用户数据 | ❌ | ✅ |
| 真实预约 | ❌ | ✅ |

### 怎么切环境

小程序代码里只改一行：

```javascript
// app.js — UAT
wx.cloud.init({ env: 'cloud1' });

// app.js — 生产（发布前改）
wx.cloud.init({ env: 'cloud-prod' });
```

**重要：** 每次发生产前，把 UAT 的测试数据清掉，或者直接在 `cloud-prod` 环境初始化干净的种子数据。

## 五、需要注意的事项

### 🔴 绝对不能做的

| 禁止 | 原因 | 正确做法 |
|------|------|------|
| 生产环境用 UAT 数据库 | 测试数据污染真实用户 | 发版前切 env |
| 生产 AppID 在 DevTools 调试 | 可能影响线上用户 | 用 UAT AppID 调试 |
| 生产环境手动改数据库 | 删错数据不可逆 | 写云函数操作 |
| SecretKey 写在前端代码 | 泄露后别人能冒充你的 App | 只放在云函数环境变量 |

### 🟡 需要注意的

| 注意点 | 说明 |
|------|------|
| API Key 共用 | 和风天气和高德 key 两个环境共用同一个，注意配额 |
| 云函数版本 | 改云函数后要在两个环境都上传部署 |
| 数据库集合结构 | UAT 和生产结构必须一致，不然代码跑不了 |
| 订阅消息模板 | 模板审核通过后两个环境都能用 |

### 🟢 可以放心的

| 项目 | 说明 |
|------|------|
| 页面代码 | UAT 和生产完全一致，不需改动 |
| 云函数代码 | 也是一致的，只需分别部署 |
| 图标/样式 | 无需任何改动 |

## 六、发布检查清单

每次从 UAT 提审生产前确认：

- [ ] cloud1 上核心流程跑通（约钓/预约/渔获）
- [ ] checklist.md 全部通过
- [ ] 将 app.js 的 env 从 `cloud1` 改为 `cloud-prod`
- [ ] `cloud-prod` 数据库集合结构确认与 `cloud1` 一致
- [ ] `cloud-prod` 已初始化种子数据（至少 3 条拼船 + 船长）
- [ ] 云函数已在 `cloud-prod` 上部署
- [ ] 隐私合规（用户协议、隐私政策已配置）
- [ ] 类目确认（体育-钓鱼）
- [ ] DevTools 上传 → 微信后台提交审核
- [ ] 审核通过后发布
- [ ] 发布后改回 `cloud1` 继续开发
