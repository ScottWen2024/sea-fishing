# 海钓通 产品需求说明书（PRD）

> 版本：v3.0 | 日期：2026-06-01 | 作者：打工虾

---

## 一、产品概述

### 1.1 产品定位

**钓友出海决策与组队工具。** 让每一次出海都不靠猜。

一句话：打开 App 就知道今天能不能钓，鱼获自动记录，找人组队一键广播。

### 1.2 目标用户

| 用户类型 | 特征 | 核心需求 |
|------|------|------|
| 新手钓客 | 0-3 次出海经验，不知道带什么、怎么钓 | 安全须知、装备推荐、导钓、识鱼 |
| 异地钓客 | 去新城市想钓鱼，不知道找谁 | GPS 定位最近港口、找本地船长 |
| 休闲钓友 | 周末出海，组队凑人 | 约钓广播、拼船接龙 |
| 资深钓手 | 有固定船长，偶尔拼人 | 天气决策、渔获记录、大鱼排行 |

### 1.3 竞品分析

| 竞品 | 类型 | 我们的差异化 |
|------|------|------|
| 懂鱼天气 | 天气+潮汐小程序 | 我们有约钓+记录+知识，不只是工具 |
| 一起海钓 | 船长撮合 App（北京钓悦） | 他们有船没导钓，我们有新手全链路 |
| 钓之趣 | 地图小程序 | 我们有约钓组队闭环 |
| 海钓e港 | 官方平台（大连） | 我们覆盖广东沿海 |

## 二、用户痛点（280 条中提炼的核心）

### P0 — 出海前
- 不知道今天能不能出海（缺少综合钓鱼指数）
- 潮汐表看不懂
- 拼船凑不够人
- 找不到靠谱船长
- 不知道带什么装备
- 新手不知道找谁带

### P1 — 出海时
- 晕船不知道怎么防
- 钓上来不认识
- 防晒不到位晒脱皮
- 深水鱼不会放气

### P2 — 出海后
- 渔获无处沉淀
- 钓多了吃不完送不完
- 被迫好评
- 同城钓友找不到

## 四、非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 首页加载 ≤ 3 秒，天气 API 超时 3 秒降级模拟数据 |
| 兼容 | H5 支持 Chrome/Safari/微信内置浏览器；小程序基础库 ≥ 2.2.3 |
| 安全 | API Key 不暴露前端，UserSig 云函数生成，_openid 隔离用户数据 |
| 数据 | 云数据库自动备份，30 分钟缓存天气数据 |
| 并发 | 云函数事务锁防超卖，单实例缓存 1000 用户共享 1 次 API 调用 |

## 七、用户旅程

### 典型一天

```
08:00  打开首页 → 看钓鱼指数 78 分 ✅推荐出海
08:01  点「📢 约钓」→ 填日期/港口 → 广播
08:05  点「🎒 装备」→ 对照清单收拾东西
08:30  出海 → 晕船了 → 点「💊 晕船指南」
12:00  钓到一条不认识 → 点「📸 识鱼」→ 石斑！
12:01  记录渔获 → 4.5 斤石斑
18:00  回家 → 打开「我的」→ 渔获市场 → 标价 ¥120
```

## 八、验收标准

| 场景 | 标准 |
|------|------|
| 天气加载 | 3 秒内显示，API 失败显示模拟数据 + 标记 |
| 发起约钓 | 提交后立即在列表中出现，其他人刷新可见 |
| 识鱼记录 | 拍照→识别→记录，3 步完成 |
| 找船预约 | 展开→筛选→选船长，3 步完成 |
| 离线可用 | 断网时页面不崩溃，显示降级数据 |
